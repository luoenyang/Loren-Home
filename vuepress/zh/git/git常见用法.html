# git 常见用法

  • 提醒小白：git push 之前一定要先 git pull ， 特别是多人协作项目！！！否则费劲！！
  • 提醒小白：git commit 不等于 git push ，二者意义不同。
  • 最好：在切换分支之后先立即 git pull ，在git push 之前再git push， 有冲突先处理冲突。

# git pull 拉取并合并

git pull = git fetch + git merge

# git fetch 拉取不合并

无特殊情况，使用git pull

# git merge / Merge 合并

如果是合并远程实际上等效于 git pull

# git commit 提交到本地

未提交前不能切换分支

# git push 提交到远程

需先提交本地，再提交远程

# resolve 释放冲突

用户解决冲突完成之后，释放冲突。 特别在push之前，解决完一定要释放。

# delete and add ignore list

把改文件、文件夹加入到git ignore

#

# git bash命令操作

它是命令行工具，用来执行git命令

# 同时推送几个分支到远程

git push <远程仓库名称> <本地分支名称1>:<远程分支名称1> <本地分支名称2>:<远程分支名称2>

git push origin branch1:branch1 branch2:branch2 git push origin branch1 branch2 名称相同时省略写

# 只提交部分文件到远程

  1. git add <文件1> <文件2> ... 把文件添加到暂存区
  2. git commit -m "描述性提交信息"
  3. git push origin <目标分支名称>

# 克隆分支

git clone [url]

# 克隆指定仓库

git clone -b [指定分支名] [远程仓库地址]

# 查看仓库状态

git status

# 添加文件到暂存区

git add

# 提交文件到版本库

git commit -m "提交信息"：将暂存区的文件提交到版本库，并附带提交信息。
git commit [file-name] -m "提交信息"：提交指定文件到版本库。

# 创建分支

git branch [branch-name]

# 查看分支，当前处于哪个分支

git branch -r：查看所有远程分支 git branch -a：查看所有本地和远程分支。

# 删除远程分支

git push origin --delete XXX

# 删除本地分支

git branch -d 会在删除前检查merge状态（其与上游分支或者与head）。
git branch -D 是git branch --delete --force的简写，它会直接强制删除。

# 新建、切换分支

git checkout [branch-name]：切换到指定分支
git checkout -b XXX local/sss 从本地其他分支拉取创建分支
git checkout -b XXX remotes/origin/SSS 从远端拉取创建分支

# 查看未暂存的变更

git diff命令可以查看未暂存的变更

# 暂存当前未提交的改动

git stash命令可以暂存当前未提交的改动

# 合并分支

git merge [branch-name]：将指定分支合并到当前分支。

# 查看远程仓库

git remote -v：查看远程仓库的详细信息，包括URL。

# 查看日志

git log

# 版本回退

git reset

# 查看仓库的操作历史

git reflog：查看仓库的操作历史，包括所有HEAD的移动记录。

# 整合分支

当你想要将一个分支的更改整合到另一个分支上，并保持一个干净的提交历史时；当你想要避免合并提交时。
git rebase

# 撤销提交

git revert
git revert是Git版本控制系统中的一个重要命令，其主要功能是撤销先前的提交，但不同于git reset的是，git revert会创建一个新的提交来撤销之前的提交，而不是直接修改或删除历史记录。

创建之后如果要推送远程，而且以后在此分支推送和拉取都默认当前分支 可使用如下命令

# -u 参数会再push的同时会指定当前分支的默认上游分支
git push -u origin 刚刚创建的分支名

# 修改当前分支提交和拉取的默认上游分支

git push -u origin 刚刚创建的分支名
或者
git branch --set-upstream-to=origin/远程分支 本地分支

# 去掉git版本控制（或者cmd操作）

find . -name ".git" | xargs rm -rf
