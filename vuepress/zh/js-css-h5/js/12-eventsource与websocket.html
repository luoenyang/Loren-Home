# eventsource 与 websocket

  • 用来与服务器建立长连接的
  • vue项目常用的热更新就是使用webpack-dev-server服务的eventsource进行热更新、热重启

# eventsource 与 websocket 的区别

区别

  • eventSource 本质仍然是 http，仅提供服务器端到浏览器端的单向文本传输，不需要心跳链接，链接断开回持续重发链接。
  • websocket 是基于 TCP 的协议，提供双向数据传输，支持二进制，需要心跳链接，断开链接不会重链。

# 什么是心跳链接

心跳链接

心跳链接：是一种机制，来检测客户端和服务端是否处于正常的链接状态。

心跳机制：跳机制是每隔一段时间会向服务器发送一个数据包，告诉服务器自己还活着，同时客户端会确认服务器端是否还活着，如果还活着的话，就会回传一个数据包给客户端来确定服务器端也还活着，否则的话，有可能是网络断开连接了，需要重连。

# eventsource
示例代码
if(typeof(EventSource)!=="undefined")
{
    var source=new EventSource("/demo/api");      //请求的url
    source.onopen=function(event);                //握手连接
    source.onerror=function(err) ;                //错误
    source.onclose=function(event) ;              //关闭
    source.onmessage=function(event){             //接收信息
        document.getElementById("result").innerHTML+=event.data + "<br>";
    };
}
else
{
    document.getElementById("result").innerHTML="抱歉，你的浏览器不支持 server-sent 事件...";
}

# nodejs 后台
var http = require("http");
var socket = require("socket.io");

// 启http服务
var server = http.createServer(function (req, res) {
  res.writeHead(200, {
    "Content-type": "text/html",
  });
  res.end();
});

server.listen("8080");
console.log("Server is running at port 8080...");

// 监听socket连接
socket.listen(server).on("connection", function (client) {
  // 接收信息
  client.on("message", function (msg) {
    // 发送消息
    client.send("hello：" + msg);
    console.log("data from client: ---> " + msg);
  });

  // 断开处理
  client.on("disconnect", function () {
    console.log("Client socket has closed.");
  });
});
