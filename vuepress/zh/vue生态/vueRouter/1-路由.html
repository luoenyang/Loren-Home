# vue-router简介

vue-router是Vue.js的官方路由管理器。它和Vue.js深度集成，用于构建单页面应用（SPA）。在单页面应用中，页面的路由和组件是紧密关联的，通过路由的变化来动态地渲染不同的组件，从而实现页面的切换。

# vue-router的核心概念

  • 路由（Route）：

路由描述了URL与UI之间的映射关系。在vue-router中，每个路由对象都包含了一个path属性（用于匹配URL）和一个component属性（用于指定当URL匹配时应该渲染的组件）。

  • 路由表（Router Table）：

路由表是一个包含多个路由对象的数组。它定义了应用中所有可能的URL与组件之间的映射关系。

  • 路由器（Router）：

路由器是vue-router的核心对象。它负责解析URL、匹配路由表、渲染对应的组件，并处理用户的导航请求。

  • 导航守卫（Navigation Guards）：

导航守卫用于在路由切换过程中执行一些额外的逻辑，如权限验证、页面跳转等。vue-router提供了全局守卫、路由独享守卫和组件内守卫等多种类型的导航守卫。

# vue-router的底层原理

  • URL与组件的映射：vue-router通过路由表来建立URL与组件之间的映射关系。当用户访问一个URL时，vue-router会解析这个URL，并在路由表中查找匹配的路由对象。一旦找到匹配的路由对象，vue-router就会渲染该路由对象指定的组件。

  • 前端路由的实现：hash模式，history模式

  • 导航守卫的执行：在路由切换过程中，vue-router会执行相应的导航守卫。

# 路由模式

# 1. hash ，如：http://www.abc.com/#/hello

  • 原理：使用 hashchange 事件来监听 hash 值的变化，从而对页面进行跳转（渲染）
// 1. hash ——即地址栏URL中的#符号
// 2. hash 值的改变，都会在浏览器的访问历史中增加一个记录。因此我们能通过浏览器的回退、前进按钮控制hash 的切换；
// 3. hash虽然出现URL中，但不会被包含在HTTP请求中，对后端完全没有影响，因此改变hash不会重新加载页面。
// 4. 网络请求不会404错误，可以配置vue路由404页面。
// 5. onhashchange事件，可以在windows对象上监听这个事件:
windows.onhashchange = function (event) {
  console.log(event.oldURL, event.newURL);
  let hash = location.hash; //通过location对象来获取hash地址
  console.log(hash); // #/notebooks/123456/list  地址从#号开始
};

    // 监听路由变化

# 2. history（传统）

利用了 HTML5 History Interface 中新增的 pushState() 和 replaceState() 方法。

  • 这两个方法应用于浏览器的历史记录站，在当前已有的 back、forward、go 的基础之上，它们提供了对历史记录进行修改的功能。
  • 这两个方法有个共同的特点：当调用他们修改浏览器历史记录栈后，虽然当前 URL 改变了，但浏览器不会刷新页面，这就为单页应用前端路由“更新视图但不重新请求页面”提供了基础。
// 1. 可使用 back,forward,go 方法
history.go(-2); //后退两次
history.go(2); //前进两次
history.back(); //后退
history.forward(); //前进

# 路由懒加载（按需加载） 、异步路由

  • 原因：打包后的文件将会异常的大，造成进入首页时，需要加载的内容过多，延时过长，不利于用户体验。
  • 操作：懒加载则可以将页面进行划分，需要的时候加载页面，可以有效的分担首页所承担的加载压力，减少首页加载用时。
  • 原理：异步加载，vue-router 配置路由 , 使用 vue 的异步组件技术 , 实现按需加载。
{ path: '/home', component: () => import('@/views/home/home.vue') } // 懒加载（按需加载路由）

# 嵌套路由

/about 路径下有一个嵌套路由 /about/user/:id。当访问 /about/user/123 时，User 组件将被渲染在 About 组件的内部。
当你在About组件内部使用 <router-view> 时，嵌套路由中的User组件将被渲染在这个位置。

// vue2
import Vue from 'vue';
import VueRouter from 'vue-router';
import Home from './components/Home.vue';
import About from './components/About.vue';
import User from './components/User.vue';
 
Vue.use(VueRouter);
 
const routes = [
  {
    path: '/',
    name: 'home',
    component: Home
  },
  {
    path: '/about',
    name: 'about',
    component: About,
    children: [
      {
        path: 'user/:id',
        name: 'user',
        component: User
      }
    ]
  }
];
 
const router = new VueRouter({
  mode: 'history',
  base: process.env.BASE_URL,
  routes
});
 
export default router;

# 简单路由实例

// vue2
import Vue from 'vue';
import VueRouter from 'vue-router';
 
// 引入组件
import HomePage from './components/HomePage.vue';
import AboutPage from './components/AboutPage.vue';
 
// 使用VueRouter
Vue.use(VueRouter);
 
// 创建路由实例
const router = new VueRouter({
  mode: 'history', // 使用HTML5 History模式
  routes: [ // 定义路由
    {
      path: '/', // 根路径
      name: 'home', // 命名路由
      component: HomePage // 对应的组件
    },
    {
      path: '/about', // 关于页面的路径
      name: 'about', // 命名路由
      component: AboutPage // 对应的组件
    }
  ]
});
 
// 创建和挂载Vue实例
new Vue({
  router // 使用路由
}).$mount('#app');
