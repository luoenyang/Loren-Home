# setup的使用

  • 作用‌：setup()是Vue3中Composition API的入口，用于配置组件。它在beforeCreate和created之前被调用，此时无法访问this。但可以通过其参数和返回值访问和暴露组件状态、属性和方法。

setup为什么没有this?

  • 一个是生命周期的问题，在beforeCreate和created之前被调用；
  • this 在选项式api中作用是访问组件的属性和方法；setup里所有的响应式都通过参数和函数的返回值传递了，都是再setup里，已经没必要this了
  • 更好的支持ts，实现类型推断，避免与ts类型系统的复杂性
  • ‌参数‌：props（只读对象，包含父组件传递的属性）、context（包含attrs、slots、emit等）
  • 返回值‌：一个对象，其属性会被合并到组件渲染上下文中，可在模板中直接使用。

# setup的好处

  • 逻辑复用‌：允许逻辑在组件间轻松复用。
  • 类型推断‌：提供更好的类型推断（ts）和自动完成。
  • 简化代码结构‌：减少冗余代码。
  • 更高效的响应式系统ref和reactive。

# 对父级暴露数据和方法defineExpose

defineExpose是一个在<script setup>中使用的函数，它的主要作用是用于显式地声明需要暴露给父组件的属性或方法。通过defineExpose，你可以指定哪些顶层变量和方法应该被暴露给外部，从而增强组件的封装性和可维护性。

查看基本用法
  • ref创建响应式可以基本数据类型也可以是对象和数组，但实际上也是用reactive来转换的，并且和基本数据类型一样在js中需要.value，需要使用.value来访问对象，在模板中则不需要.value来访问。
  • reactive创建响应式是专门用于引用数据类型的 如对象和数组，访问时直接访问，不需要使用.value

onMounted(() => {});

# 响应式变量

vue3和vue2最大的区别之一就是响应式变量

代码示例
 <script setup>
    import { ref, toRef, toRefs, reactive, onMounted } from "vue";

    // 创建任意数据类型的响应式变量，取值使用 .value 如val0.value，val1.value
    const val0 = ref(1);
    console.log("val0", val0, "value=>", val0.value);

    // 取值：val1.value
    const val1 = ref({ num: 1, num2: 2 });
    console.log("val1", val1, "value=>", val1.value);

    // 创建引用类型的响应式对象，val3等于val1.value ; 取值： val3.num
    const val3 = reactive({ num: 1, num2: 2 });
    console.log("val3", val3, "num=>", val3.num);

    // 注意：不要用这种方式，应该使用 ref
    // const val2 = reactive(2);
    // console.log("val2", val2);

    // 将响应式对象里的字段批量转换为普通对象（此对象是新的响应式变量）
    const { num } = toRefs(val3);
    console.log("num", num);

    // 将响应式对象num字段创建为一个新的响应式变量
    const val4 = toRef(val3, "num");
    console.log("val4", val4, "等于", num);
 </script>

# props 和 emit

代码示例
<script setup>
  // **** defineProps 声明 props *****

  const props = defineProps({
    dataId: {
      req
      type: String,
      required:false, // 这种写法 默认是false
      default: ''
    }
  })

// 采用ts专有声明，无默认值
   defineProps<{ 
    msg: string
    num?: number // ? 代表 num 不必须传, 不带则是必传项
    arr1?:Array<any>
    arr2?:String
    arr3?:String[]
    arr4?:any
  }>()

  // 采用ts专有声明，有默认值
  // 可先定义接口
  interface Props {
      msg?: string
      labels?: string[]
  }
  const props = withDefaults(defineProps<Props>(), {
      msg: 'hello',
      labels: () => ['one', 'two']
  })




  // **** defineEmits 抛出事件 *****

  // 声明 emit 事件，事先需要声明好事件名，如on-update
  const emit = defineEmits(['on-update'])
  // ts 写法
  const emit = defineEmits<{
    (e: 'click', id: number): void
    (e: 'change', id: number): void
    (e: 'on-update', value: number): void
  }>()
  const onUpdate = () => {
    // 执行 on-update 事件
    emit('on-update', 2)
  }



  // **** defineExpose 子组件暴露自己的属性  *****

  // 非setup写法 只要是在data里定义的父组件都可以获取到
  defineExpose({
    count
  })
  // 父组件调用见 ref
