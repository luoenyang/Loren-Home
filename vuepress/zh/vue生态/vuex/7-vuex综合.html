# 一. 什么是 vuex?

它是应用程序开发的状态管理模式，集中管理组件状态。

每一个 Vuex 应用的核心就是 store（仓库）。“store”基本上就是一个容器，它包含着应用中大部分的状态 (state)。

# 二. Vuex 和单纯的全局对象：

2 点不同

  1. Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。
  2. 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是使用.commit显式地提交 mutation。这样使得我们可以方便地跟踪每一个状态的变化，从而让我们能够实现一些工具帮助我们更好地了解我们的应用。

# 创建 getters
const store = new Vuex.Store({
  state: {
    todos: [
      { id: 1, text: "...", done: true },
      { id: 2, text: "...", done: false },
    ],
  },
  getters: {
    doneTodos: (state) => {
      return state.todos.filter((todo) => todo.done);
    },
  },
});

# 3. mutation : 非常类似于事件

  1. 最好提前在你的 store 中初始化好所有所需属性; 即在 state 中定义好相关属性
  2. 当需要在对象上添加新属性时，你应该 Vue.set(obj, 'newProp', 123)
  3. mutations 中 必须是同步函数,不能是异步函数,异步使用 actions
# 创建
const store = new Vuex.Store({
  state: {
    count: 1,
  },
  mutations: {
    increment(state, obj) {
      // 变更状态
      state.count += obj.amount;
    },
  },
});

# 4. actions: 类似于 mutation , actions 中是异步函数

  • Action 提交的是 mutation，而不是直接变更状态,
  • Action 可以包含任意异步操作。
  • 在组件中使用 this.$store.dispatch('incrementAsync') 触发
// 创建
actions: {
  incrementAsync ({ commit },obj) {
    setTimeout(() => {
      commit('increment')
    }, 1000)
  }

  // async/await 组合action 很常用
  async actionA ({ commit }) {
    commit('gotData', await getData())
  },
  async actionB ({ dispatch, commit }) {
    // 等待 actionA 完成
    await dispatch('actionA')
    commit('gotOtherData', await getOtherData())
  },

  actionC ({ commit },obj) {
    return new Promise((resolve, reject) => {
      setTimeout(() => {
        commit('someMutation')
        resolve()
      }, 1000)
    })
  }
}

# 5. modules: 分拆 store

  • 由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿。
  • 为了解决以上问题，Vuex 允许我们将 store 分割成模块（module）。
  • 每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块
const moduleA = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

const moduleB = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}


const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

// 或者
const store = new Vuex.Store({

})

store.registerModule('myModuleA', moduleA);
store.registerModule('myModuleB', moduleB);



store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

# 四. 完整使用方法

见 store , main.js , index.vue

  • 由于 store 中的状态是响应式的，在组件中调用 store 中的状态简单到仅需要在计算属性中返回即可。
  • 触发变化也仅仅是在组件的 methods 中提交 mutation。

# 1. 创建 :在 store 文件夹下 index 文件夹中 index.js: 组装模块并导出 store 的地方

import { createStore } from "vuex";

const store = createStore({
  state: {
    count: 0,
    todos: [
      { id: 1, text: "...", done: true },
      { id: 2, text: "...", done: false },
    ],
  },
  getters: {
    doneTodos: (state) => {
      return state.todos.filter((todo) => todo.done);
    },
  },
  mutations: {
    addCount(state) {
      state.count++;
    },
    increment(state, obj) {
      // 变更状态
      state.count += obj.amount;
    },
  },
  actions: {
    // async/await 组合action 很常用
    async actionA({ commit }) {
      commit("gotData", await getData());
    },
    async actionB({ dispatch, commit }) {
      // 等待 actionA 完成
      await dispatch("actionA");
      commit("gotOtherData", await getOtherData());
    },

    actionC({ commit }) {
      return new Promise((resolve, reject) => {
        setTimeout(() => {
          commit("someMutation");
          resolve();
        }, 1000);
      });
    },
  },
});

export default store;

# 3. 组件中使用

import { mapState , mapGetters , mapActions , mapMutations} from "vuex";
export default {
    computed:{
      // 使用对象展开运算符将 mapState 混入 computed 对象中
      ...mapState({
          count: state=> state.count,
      })
      // 使用对象展开运算符将 getter 混入 computed 对象中
      ...mapGetters([
        'doneTodos',
      ])
    },
    methods:{
      ...mapMutations([
        'increment',              // 将 `this.increment(obj)` 映射为 `this.$store.commit('increment',obj)`
      ]),
      ...mapMutations({
        doIncrement: 'increment'  // 将 `this.doIncrement(obj)` 映射为 `this.$store.commit('increment',obj)`
      })

      ...mapActions([
        'actionC',                // 将 `this.actionC(obj)` 映射为 `this.$store.dispatch('actionC',obj)`
      ]),
      ...mapActions({
        doActionC: 'actionC'      // 将 `this.doActionC(obj)` 映射为 `this.$store.dispatch('actionC',obj)`
      })
    },
    mounted(){
      // 可从状态中获取 count
      console.log(this.count)
    },
}
