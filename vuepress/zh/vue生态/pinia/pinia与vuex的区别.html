# pinia与vuex区别

# pinia优点：

  • 热模块更换，借助了 import.meta
  • 插件：使用插件扩展 Pinia 功能
  • 服务端渲染（建议：nuxt.js）
  • 提供适当的 TypeScript 支持或 autocompletion

# pinia与vuex区别：

  • 不再有 mutations ，而是使用 actions 来代替 -- 相当于组件中的 methods。
    • vuex 中 actions 是服务 mutations，[vuex actions 链接]
  • 不再有 modules 的嵌套结构。[vuex modules 链接]
  • 没有 命名空间模块。
  • 无需动态添加 Store，默认情况下它们都是动态的。
  • 无需创建自定义复杂包装器来支持 TypeScript，所有内容都是类型化的，并且 API 的设计方式尽可能利用 TS 类型推断。
# 总结：pinia更简洁了。API语法大致相同 。
