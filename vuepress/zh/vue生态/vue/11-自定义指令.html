# 钩子函数

钩子

  1. bind：只调用一次，指令第一次绑定到元素时调用。在这里可以进行一次性的初始化设置。此时未插入到DOM。
  2. inserted：被绑定元素插入父节点时调用。此时元素被插入到DOM。
  3. update：所在组件的 VNode 更新时调用。
  4. componentUpdated：指令所在组件的 VNode 及其子 VNode 全部更新后调用。
  5. unbind：只调用一次，指令与元素解绑时调用。
# 钩子函数参数: el参数外,其它都只读,勿修改

注意

  1. el：指令所绑定的元素，可以用来直接操作 DOM。

  2. binding：一个对象，包含以下 property：

    name：指令名，不包括 v- 前缀。
    value：指令的绑定值，例如：v-my-directive="1 + 1" 中，绑定值为 2。
    oldValue：指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。
    expression：字符串形式的指令表达式。例如 v-my-directive="1 + 1" 中，表达式为 "1 + 1"。
    arg：传给指令的参数，可选。例如 v-my-directive:foo 中，参数为 "foo"。
    modifiers：一个包含修饰符的对象。例如：v-my-directive.foo.bar 中，修饰符对象为 { foo: true, bar: true }

  3. vnode：Vue 编译生成的虚拟节点.

  4. oldVnode：上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用。

Vue.directive('demo', {
  bind: function (el, binding, vnode) {
    var s = JSON.stringify
    el.innerHTML =
      'name: '       + s(binding.name) + '<br>' +
      'value: '      + s(binding.value) + '<br>' +
      'expression: ' + s(binding.expression) + '<br>' +
      'argument: '   + s(binding.arg) + '<br>' +
      'modifiers: '  + s(binding.modifiers) + '<br>' +
      'vnode keys: ' + Object.keys(vnode).join(', ')
  }
})

# 自定义指令作用，使用场景

  1. 当需要对普通 DOM 元素进行底层操作，此时就会用到自定义指令
  • 自动聚焦
  • 权限控制
  • 封装点击隐藏（click-outSide）
  1. 但是，对于大幅度的 DOM 变动，还是应该使用组件

# 1. 使用示例 click-outSide 的全局自定义指令

vue2
Vue.directive('click-outside', {  
  bind: function (el, binding, vnode) {  
    // 保存原始点击事件处理器（如果有的话）  
    el.__vueClickOutside__ = function (event) {  
      // 检查点击事件的目标是否是el或其子元素  
      if (!(el === event.target || el.contains(event.target))) {  
        // 如果不是，则调用绑定的值（应该是一个函数）  
        binding.value(event);  
      }  
    };  
  
    // 在document上添加事件监听器  
    document.body.addEventListener('click', el.__vueClickOutside__);  
  },  
  unbind: function (el) {  
    // 移除事件监听器  
    document.body.removeEventListener('click', el.__vueClickOutside__);  
    // 清理保存的处理器  
    delete el.__vueClickOutside__;  
  }  
});

# 3. 前端面试： vue 如何设计一个权限控制的自定义指令，使用了函数柯里化

  • 确定需求 首先，明确权限控制的需求。例如，根据用户的角色或权限级别来决定是否显示某个元素或组件隐藏显示
  • 设计指令 使用 Vue.directive或组件内的directives选项来定义(如上局部指令)。
  • 实现指令
    1. 指令名称：选择一个有意义的名称，如v-permission。
    2. 钩子函数：根据需要实现bind、inserted、update等钩子函数。对于权限控制，bind或inserted通常足够。
    3. 权限检查逻辑：在指令中实现权限检查逻辑。这可能涉及从Vuex、Vue Router或其他状态管理库中获取当前用户的权限信息。
    4. DOM操作：根据权限检查结果，对DOM元素进行隐藏、禁用或其他操作。
查看自定义指令
// 假设你有一个Vuex store来管理用户权限
// store.js
const store = new Vuex.Store({
  state: {
    userPermissions: ['admin', 'editor'] // 示例权限列表
  },
  getters: {
    // 函数柯里化技巧，在此，在Vuex的getter的实际使用中，您不需要手动传入state，因为Vuex会自动为您处理这部分
    // Vuex的getter中，state是作为第一个参数自动传入的
    //  什么是函数柯里化：JavaScript中的函数柯里化（Currying）的一个例子。柯里化是将一个多参数的函数，转换成一系列使用一个参数的函数的技术。通过这种方式，你可以创建一个接受一个参数并返回另一个函数的函数，这个返回的函数再接受下一个参数，依此类推，直到所有参数都被接收，并返回最终结果。
    hasPermission: (state) => (permission) => state.userPermissions.includes(permission)
  }
});

// 自定义指令 v-permission
Vue.directive('permission', {
  // 指令绑定到元素时调用
  bind(el, binding, vnode) {
    // 从Vuex store中获取hasPermission getter
    const hasPermission = store.getters.hasPermission;

    // 检查当前用户是否有指定的权限
    if (!hasPermission(binding.value)) {
      // 如果没有权限，则隐藏元素（或进行其他操作，如禁用）
      el.style.display = 'none';
    }
  },
  // 当绑定元素更新时调用（可选）
  update(el, binding) {
    // 这里可以添加额外的更新逻辑，但通常权限不会在运行时改变
  }
});

// 使用指令的Vue组件
new Vue({
  el: '#app',
  store,
  template: `
    <div>
      <button v-permission="'admin'">Only Admins Can See This</button>
      <button v-permission="'editor'">Editors and Admins Can See This</button>
      <button>Everyone Can See This</button>
    </div>
  `
});
