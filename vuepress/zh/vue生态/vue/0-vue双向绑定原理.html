# Object.defineProperty

原生实现-双向数据绑定的简单实现，简单的vue
原生实现-双向数据绑定的实现，数据同步的另一种实现

Object.defineProperty(obj, prop, descriptor) 的定义

  • Object.defineProperty()的作用就是直接在一个对象上定义一个新属性，或者监听、修改一个已经存在的属性。

三个参数

  • obj：要定义属性的对象
  • prop：要定义或修改的属性的名称
  • descriptor：要定义或修改的属性描述符，是个对象

descriptor 属性描述符的 5 个属性

  • value 属性对应的值
  • writable 是否可以改变
  • get 该方法返回值被用作属性值
  • set 该方法将接受唯一参数，并将该参数的新值分配给该属性
  • configrable 描述属性是否配置，以及可否删除
  • numerable 描述属性是否会出现在for in 或者 Object.keys()的遍历中 （即：是否可枚举）

使用：两种形式

通过Object.defineProperty()为对象定义属性，有两种形式，且不能混合使用，分别为

  • 数据描述符
  • 存取描述符
  • 数据描述符 的使用形式
let Person = {}
Object.defineProperty(Person, 'name', {
  configrable:true,
  numerable:true,
  value:'luo',
  writable:true,
}

# 原理

原理

  1. vue数据双向绑定是通过数据劫持结合发布者-订阅者模式的方式来实现的。
  2. Object.defineProperty()对属性的劫持, 对其get和set进行重写操作。

# 过程

过程

  1. 我们已经知道实现数据的双向绑定，首先要对数据进行劫持监听，所以我们需要设置一个监听器Observer，用来监听所有属性。

  2. 如果属性发上变化了，就需要告诉订阅者Watcher看是否需要更新。因为订阅者是有很多个，所以我们需要有一个消息订阅器Dep来专门收集这些订阅者，然后在监听器Observer和订阅者Watcher之间进行统一管理的。

  3. 接着，我们还需要有一个指LING解析器Compile，对每个节点元素进行SHAO描和解析，将相关指令对应初始化成一个订阅者Watcher，并替换模板数据或者绑定相应的函数，此时当订阅者Watcher接收到相应属性的变化，就会执行对应的更新函数，从而更新视图。

总结：订阅者Watcher是监听器和解析器的一个桥梁

以下3个步骤，实现数据的双向绑定：

1.实现一个监听器Observer，用来劫持并监听所有属性，如果有变动的，就通知订阅者。

2.实现一个订阅者Watcher，可以收到属性的变化通知触发解析器Compile并执行相应的函数，从而更新视图。

3.实现一个解析器Compile，解析 Vue 模板指令，将模板中的变量都替换成数据。可以扫描和解析每个节点的相关指令，并根据初始化模板数据以及初始化相应的订阅器。

# 扩展：vue框架对数组或者对象的监听

如何监听：通过遍历数组 和递归遍历对象，利用 Object.defineProperty() 对每个属性进行监听。

注意：

  1. 通过索引直接设置一个数组项时是监听不到的，如：items[indexOfItem] = newValue，解决办法：vue.set()
  2. 修改数组长度也是监听不到的，如：items.length，解决办法：Array.prototype.splice

# vm.$set()原理

    1. 先判断是数组还是对象，是数组的用 push,shift,pop,splice,unshift,sort,reverse 方法触发响应式。
    1. 如果是对象，还要判断是对象，采用 Object.defineProperty， 动态添加 getter 和 setter 。
