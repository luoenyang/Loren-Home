# 1. 模板实现原理（vue组件的渲染过程）

原理

首先将模板转化为ast语法树，进行静态节点标记，主要用来做虚拟DOM的渲染优化（优化器），语法树再通过generate转化为render函数，再通过watcher监测， 当属性发生变化时调用render函数生成VNode对象, vNode通过createElement方法生成虚拟dom，再通过patch方法, DOM Diff算法渲染成真实dom

  1. 模板 - 语法树 - render函数 - watcher
  2. watcher - render函数 - vnode对象 - 虚拟dom - dom diff 算法 - 渲染成真实dom

# vue组件的渲染过程

  • 解析模板-抽象成语法树（ast）
  • 将语法树转为渲染函数，以生成虚拟节点（vNode）
  • 使用渲染函数进行虚拟DOM的渲染，再进行Diff算法进行与真实DOM的比对
  • 最后进行真实DOM更新

# 2. 浏览器渲染流程

1. 用HTML分析器创建DOM树
2. 用CSS分析器生成样式表
3. 将DOM树和样式表，关联起来，构建一颗Render树  
4. 浏览器进行布局Layout 
5. Render树和节点显示坐标都有了，就调用每个节点paint方法，把它们绘制出来。

# 3. dom diff 算法

定义：建虚拟DOM并将其映射成真实DOM

作用：某些操作的时候可能会出现需要更新某一个dom元素，不更新整个组件，使用diff算法配合虚拟dom即可实现局部dom更新。

过程：当数据变化后(setState())，会重新根据新的数据，创建新的虚拟DOM对象（树）与上一次得到的虚拟DOM对象，使用Diff算法比对（找不同），得到需要更新的内容.

vue 的 diff 优势：只进行同层级比较，忽略跨级操作

# 虚拟DOM与真实DOM的区别

注意：需不需要虚拟DOM，其实与框架的DOM操作机制有关

    1. 虚拟DOM不会进行排版与重绘操作,本质上就是一个JS对象，用来描述你希望在屏幕上看到的内容
    1. 虚拟DOM进行频繁修改，然后一次性比较并修改真实DOM中需要改的部分，最后并在真实DOM中进行排版与重绘，减少排版与重绘损耗
    1. 真实DOM频繁排版与重绘的效率是相当低的

# vue中 key值得作用

    1. 可以唯一的确定一个DOM元素，让diff算法更加高效(更新虚拟dom和真实dom对比)
    1. 高效的更新虚拟 DOM，提高性能
    1. vue在使用相同的标签名元素的过渡切换时，也会使用到key属性，其目的也是为了让vue可以区分它们，否则vue只会替换其内部属性而不会触发过渡效果。
    1. 另外，若不设置key，还可能在列表更新时引发一些隐蔽的bug(如列表删除某项)
