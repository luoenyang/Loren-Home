# 父子组件生命周期执行顺序

# vue组件的渲染过程

  • 解析模板-抽象成语法树（ast）
  • 将语法树转为渲染函数，以生成虚拟节点（vNode）
  • 使用渲染函数进行虚拟DOM的渲染，再进行Diff算法进行与真实DOM的比对
  • 最后进行真实DOM更新

# 5. Proxy 与 Object.defineProperty

# Proxy 的优势如下:

Proxy 对象是ES6新出的一个特性,用于创建一个对象的代理,从而实现基本操作的拦截和自定义(如属性查找、赋值、枚举、函数调用等)。

    1. Proxy 可以直接监听对象而非属性；
    1. Proxy 可以直接监听数组/对象的变化，删除和添加；
    1. Proxy 有多达 13 种拦截方法,不限于 apply、ownKeys、deleteProperty、has 等等是 Object.defineProperty 不具备的；
    1. Proxy 返回的是一个新对象,我们可以只操作新的对象达到目的,而 Object.defineProperty 只能遍历对象属性直接修改；
    1. Proxy 作为新标准将受到浏览器厂商重点持续的性能优化，es6特性；

# 6. vue3特性

    1. 基于代理 Proxy 的 observer 实现，替代Object.defineProperty ，看上面
    1. 作用域插槽，2.x 的作用域插槽变了，父组件会重新渲染，而 3.0 把作用域插槽改成了函数的方式，只会影响子组件的重新渲染。
    1. vue3.0 也全面改用 TypeScript 来重写了，更是使得对外暴露的 api 更容易结合 TypeScript。

# defineComponent 作用：

0.defineComponent函数，只是对setup函数进行封装，返回options的对象；
1.defineComponent 可以给组件的setup方法准确的参数类型定义,引入 defineComponent() 以正确推断 setup() 组件的参数类型
2.defineComponent 可以接受显式的自定义 props 接口或从属性验证对象中自动推断
3.defineComponent 可以正确适配无 props、数组 props 等形式
4.defineComponent最重要的是：在TypeScript下，给予了组件 正确的参数类型推断
