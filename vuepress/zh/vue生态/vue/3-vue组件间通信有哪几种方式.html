# vue 组件间通信有哪几种方式？(6大种)

  • 1、vuex 应用程序开发的状态管理模式。每一个 Vuex 应用的核心就是 store（仓库）。

详见vuex综合。

  • 2、props / v-on / $emit：适用 父子组件通信，详见 4-vue 父子通信.md

详见 4-vue父子通信

  • 3、$ref：适用 访问子组件/子元素
  • 4、root / parent / children：适用 父子组件通信

  • 5、provide / inject： 适用于 隔代组件通信 ，使用场景：在子孙组件控制父的属性
// 父组件
export default {
  // 通过 provider 来提供变量，然后在子孙组件中通过 inject 来注入变量，进行通信
  provide() {
    return {
      // 在父组件提供变量
      reload: this.reload,
    };
  },
  methods: {
    reload() {},
  },
};


// 子孙组件
export default {
  // 子孙组件注入变量
  inject: ["reload"],
  methods:{
      todo(){
          // 子孙组件使用注入的变量
          this.reload()
      }
  }
}

  • 6、$attrs / inheritAttrs / $listeners ：适用于 隔代组件通信
<!-- 使用$attrs ，inheritAttrs 属性 能够使用简洁的代码，将 A 组件的数据传递给 C 组件；-->
<!-- 在 B 组件上 绑定 v-on=”$listeners”, 在 A 组件中，监听 C 组件触发的事件。 -->

<!--A组件-->
<template>
  <div>
    <child name="名字" @change="change"></child>
  </div>
</template>

<script>
export default {
  methods: {
    change() {
      alert("孙组件传出来的时间");
    },
  },
};
</script>

<!--B组件-->
<template>
  <div>
    <!-- 绑定 $attrs 和 $listeners -->
    <grand-child v-on="$attrs" v-bind="$listeners"></grand-child>
  </div>
</template>

<script>
export default {
  // vue官网对于inheritAttrs的属性解释：如果你不希望组件的根元素继承特性，你可以在组件的选项中设置 inheritAttrs: false。
  // 1.当设置inheritAttrs: true（默认）时，子组件的顶层标签元素中（本例子的div元素）会渲染出父组件传递过来的属性
  // 2.当设置inheritAttrs: false时，子组件的顶层标签元素中（本例子的div元素）不会渲染出父组件传递过来的属性
  // 3.不管inheritAttrs为true或者false，子组件中都能通过$attrs属性获取到父组件中传递过来的属性。
  // 而此方法正式使用$attrs传递
  inheritAttrs: false, // 这里设置为false是因为目的是把参数传到C组件，B组件不需要
};
</script>

<!--C组件-->
<template>
  <div>
    名字：{{ name }}
    <div @click="btn">点击事件</div>
  </div>
</template>

<script>
export default {
  props: {
    // 对应A组件的name，实现隔代通信
    name: {
      type: String,
      default: "",
    },
  },
  methods: {
    btn() {
      // 会触发 A 组件的 change 方法
      this.$emit("change");
    },
  },
};
</script>
