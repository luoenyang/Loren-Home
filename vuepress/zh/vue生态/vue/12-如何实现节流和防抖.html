节流与防抖

# 什么是节流?

高频事件触发，但在n秒内函数只会执行一次，所以节流会稀释函数的执行频率。
节流更关注事件触发的频率。

  • 使用场景（按需使用防抖还是节流）
  • 页面滚动加载
  • 触摸滑动
  • 拖拽和平移
  • API请求速率限制
  • 搜索建议
  • 表单验证
function throttle(func, wait) {
    let timeout = null;
    let lastRun = 0; // 上次执行函数的时间戳

    return function(...args) {
        const now = new Date().getTime(); // 当前时间戳

        // 如果当前时间距离上次执行函数的时间超过了wait，则允许执行函数
        if (now - lastRun >= wait) {
            if (timeout !== null) {
                clearTimeout(timeout); // 清除之前的定时器
                timeout = null;
            }

            func.apply(this, args); // 执行原函数，并传入当前上下文和参数
            lastRun = now; // 更新上次执行函数的时间戳
        } else if (timeout === null) {
            // 如果当前时间距离上次执行函数的时间还没有超过wait，并且之前没有设置定时器
            // 则设置一个定时器，在wait时间后执行函数（这里其实是为了处理在节流间隔内再次触发事件的情况）
            timeout = setTimeout(function() {
                lastRun = new Date().getTime(); // 更新上次执行函数的时间戳为当前时间
                func.apply(this, args); // 执行原函数，并传入当前上下文和参数（注意这里的this指向可能不正确，通常需要使用闭包或箭头函数来保持正确的this指向）
                timeout = null; // 清除定时器，表示已经执行过一次了
            }.bind(this), wait - (now - lastRun)); // 计算剩余时间并设置定时器
        }
        // 注意：这里的else if分支在某些节流实现中可能会被省略，因为即使设置了定时器，
        // 在wait时间后执行函数时也会更新lastRun，从而避免在定时器执行期间再次触发事件导致的重复执行。
        // 但是，保留这个分支可以确保在极端情况下（如函数执行时间很长，接近或超过wait时间）的正确性。
    };
}

// 使用示例
const throttledFunction = throttle(() => {
    console.log('Function executed at', new Date().toISOString());
}, 2000); // 每2秒最多执行一次

// 模拟频繁触发事件
setInterval(throttledFunction, 500); // 每0.5秒尝试触发一次，但实际上每2秒才会执行一次

# 什么是防抖?

触发高频事件后n秒内函数只会执行一次，如果n秒内高频事件再次被触发，则重新计算时间。
防抖更关注事件的结束状态。

  • 使用场景（按需使用防抖还是节流）
  • 输入框的输入事件：避免用户每次输入都触发处理函数。
  • 窗口调整大小事件：避免在窗口大小变化时频繁执行布局调整。
  • 滚动事件：避免滚动时频繁执行滚动处理函数。
  • 按钮点击事件：防止双击提交
// 防抖
function debounce(func, wait) {
    let timeout;

    return function(...args) {
        const context = this;

        clearTimeout(timeout);
        timeout = setTimeout(() => func.apply(context, args), wait);
    };
}

// 使用示例
const handleResize = () => {
    console.log('Window resized at', new Date().toLocaleTimeString());
};

window.addEventListener('resize', debounce(handleResize, 300));

# 总结：

总结

函数防抖：将多次操作合并为一次操作进行。原理是维护一个计时器，规定在delay时间后触发函数，但是在delay时间内再次触发的话，就会取消之前的计时器而重新设置。这样一来，只有最后一次操作能被触发。

函数节流：使得一定时间内只触发一次函数。原理是通过判断是否有延迟调用函数未执行。

区别

区别： 关键:防抖是重新计算时间

  1. 函数节流不管事件触发有多频繁，都会保证在规定时间内一定会执行一次真正的事件处理函数
  2. 而函数防抖只是在最后一次事件后才触发一次函数。 比如在页面的无限加载场景下，我们需要用户在滚动页面时，每隔一段时间发一次 Ajax 请求，而不是在用户停下滚动页面操作时才去请求数据。这样的场景，就适合用节流技术来实现。
