# 1. watch 监听属性

# 参数

handler:(){

}
immediate: true, // 初始化watch时（Watcher是一个负责更新）就立即执行handler回调函数
deep: true,      // 递归监听对象的属性

# 过程

在created函数调用之前，调用了initWatcher方法，传入参数，为每一个watcher属性实例化了一个Watcher，实例化的结尾会调用watcher prototype.get方法，该方法会获得值并返回，get函数执行即导致了watcher被收集为依赖，至此成功的监听了属性。 get执行时如果deep为真，则会递归监听所有的属性。

即

beforeCreated -> initWatcher -> 重写prototype.get -> 返回给 watcher -> 成功监听

# 用法

watch: {
    taskList: {
        handler(newValue, oldValue) {
            console.log(newValue)
        },
        deep: true,
        immediate: true
    }
}

# 扩展： js 原生 es5的 object.defineProperty

var inner = { first:1111 }
var test={    name:inner  }

Object.defineProperty(test,"name",{
    get(){         
        console.log("name get被触发")         
        return inner
    },
    set(){        
        console.log("name set被触发")
    }

// 访问 test.name 第一次，触发 name 的 get
Object.defineProperty(test.name,"first",{
    get(){        
        return console.log("first get被触发")
    },
    set(){        
        console.log("first set被触发")
    }
})

// 访问 test.name 第二次，触发 name 的 get
var a = test.name
// 独立访问 first 第一次
var b= a.first 
// 独立访问 first 第二次
b= a.first
// 独立改变 first
a.first = 5

# 2. computed 计算属性

computed

  1. computed是计算属性，watch是监听，监听data中某个属性的变化。
  2. computed支持缓存，依赖的属性值发生变化就会重新计算，否则使用缓存的值。
  3. computed第一次加载就执行一次，而watch是监听的属性数值产生变化后才执行。
  4. computed内要用return（即get方式）进行返回。

# vue compute中 set 方法的作用

  • 作用：当计算属性被赋值时（set使得计算属性可以被赋值），你可以通过 set 方法来定义如何响应这个赋值操作，即如何更新计算属性所依赖的数据。
  • 使用场景：双向绑定的复杂计算逻辑。

解析：在这个例子中，当你尝试设置fullName的值，比如vm.fullName = 'John Doe'，setter方法将被调用，并且会把新的全名分割成名和姓，然后更新firstName和lastName。这样，计算属性fullName的值就会依据这两个响应式数据的新值重新计算。

computed: {
    fullName: {
      // getter 方法
      get() {
        return `${this.firstName} ${this.lastName}`;
      },
      // setter 方法
      set(value) {
        const names = value.split(' ');
        this.firstName = names[0];
        this.lastName = names[names.length - 1];
      }
    }
}

# 3. 计算属性和监听属性的区别

区别

  1. 计算属性变量在computed中定义，属性监听在data中定义。

  2. 计算属性是声明式的描述一个值依赖了其他值，依赖的值改变后重新计算结果更新DOM。属性监听的是定义的变量，当定义的值发生变化时，执行相对应的函数。

# 4. 使用场景

computed 　　　　
　　　　当一个属性受多个属性影响的时候就需要用到computed
　　　　最典型的例子： 购物车商品结算的时候
watch
　　　　当一条数据影响多条数据的时候就需要用watch
　　　　搜索数据

# 5. vue2而言，少使用watch, 直接监听事件

原因

  1. 如果watch的是一个对象，reference equal的变化不会触发watch回调，这里包括： -属性增减（delete 或直接新增属性） -修改某一个属性的值（需要添加deep: true的选项）

  2. 如果watch的对象赋值给一个内部所有属性和属性的值相等的新对象，也会触发更新。

  3. Vue内部已经重写了数组的一些方法，如： splice, shift, push pop等，但下面的情况不能捕获更新： -直接用index去更新数组，如 cities[1] = 'sss' -直接修改cities的长度 如cities.length = 0
