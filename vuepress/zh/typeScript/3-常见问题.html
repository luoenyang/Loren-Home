# 1. TypeScript优势

见 1-什么是TypeScript

  • typeScript 是 JavaScript 的超集，为 JavaScript 添加了静态类型和其他面向对象编程的特性。

  • 主要区别在于 TypeScript 具有静态类型系统，使得在开发过程中能更早地发现类型错误（编译时而不是运行时），增强了代码的可维护性和可读性。

  • ‌TypeScript在JavaScript的基础上新增了以下数据类型‌：

  • Void‌：表示没有返回值，通常用于函数返回类型。

  • ‌Any‌：表示任意类型，可以接受任何类型的值。

  • ‌Never‌：表示永远不会有的值，通常用于表示错误情况或永远不会执行到的代码块。

  • ‌Tuple‌：表示元组类型，是一种限制数组的元素类型和个数的数组。

  • ‌Enum‌：表示枚举类型，用于定义一组命名的常量。

# ts 的内置类型（内置工具函数）
  • 基本类型：string,number,boolean,null,undefined,bigInt,viod:
  • 引用类型（复合类型）：Object,ArrAy<T>,Tuple[T1,T2...],enum
  • 工具类型（工具函数）：
  1. Partial<T>‌：将类型T中的所有属性变为可选的。例如，Partial<User>会将User接口中的所有属性变为可选的‌
  2. Required<T>‌：将类型T中的所有属性变为必需的。例如，Required<Props>会将Props接口中的所有属性变为必需的‌
  3. Readonly<T>‌：将类型T中的所有属性变为只读的。例如，Readonly<User>会将User接口中的所有属性变为只读‌
  4. Pick<T, K>‌：从类型T中选择指定的K属性集合。例如，Pick<User, ['name', 'age']>会选择User接口中的name和age属性‌
  5. Record<K, V>‌：创建一个类型，其属性键由类型K组成，对应值类型为V。例如，Record<string, number>会创建一个键为字符串、值为数字的对象类型‌
  6. ‌Exclude<T, U>‌：从类型T中排除所有U类型的属性。例如，Exclude<string, number>会排除所有数字类型的属性‌
  7. Extract<T, U>‌：从联合类型T中提取出U类型的成员类型。例如，Extract<string | number, string>会提取出字符串类型的成员‌
  8. Omit<T, K>‌：创建一个类型，与T类似，但排除了K中列出的属性。例如，Omit<User, 'age'>会创建一个没有age属性的User接口‌
  9. ‌Parameters<T>‌：提取函数类型T的参数类型。例如，Parameters<() => void>会提取出该函数的参数类型‌
  10. ReturnType<T>‌：提取函数类型T的返回类型。例如，ReturnType<() => string>会提取出该函数的返回类型‌
interface Person {
   name: string;
   age: number;
   address: string;
   phone: string;
}

// 创建一个类型，与Person类似
type PersonWithoutAddressAndPhone = Omit<Person, 'address' | 'phone'>; // Omit<Person, keyof Person>

// PersonWithoutAddressAndPhone 类型为：
// {
//    name: string;
//    age: number;

# 2. TypeScript中const和readonly的区别是什么？

  • const作用于变量（常量），readonly作用于属性（只读属性）
  • 当const变量保存的数组，可以使用push，pop等方法。但是如果使用ReadonlyArray<>声明的数组不能使用push，pop等方法。
const arr:Array<number>= [1, 2, 3];
// 允许
// arr.push(4);

const arr2:ReadonlyArray<number>= [1, 2, 3];
// 不允许
// arr2.push(4);

# 3.TypeScript中any,never,unknown和viod有什么区别？

    1. any：变量或者属性可以是任意数据类型，接受任意类型赋值。
    1. unknown类型和any类型类似。与any类型不同的是unknown类型可以接受任意类型赋值，但是unknown类型赋值给其他类型前，必须被断言。
    1. never，never表示永远不存在的类型。比如一个函数总是抛出错误，而没有返回值。或者一个函数内部有死循环，永远不会有返回值。函数的返回值就是never类型。
    1. void, 没有返回值（undefined，null）的类型。运行时处于高优先级。
// 在 TypeScript 中，当你有一个 unknown 类型的值时，不能直接将其赋值给另一个具体类型的变量。
// 你需要使用类型断言或类型保护来确保类型安全。类型断言是一种直接的方式，但应该谨慎使用，因为它可能会绕过 TypeScript 的类型检查。
// 类型保护则是一种更安全的方式，通过条件检查来确保值的类型

let value: unknown;  

// 类型断言

// 假设我们知道 value 实际上是 number 类型  
let numberValue = value as number;  // 类型断言

// 或者

// 类型保护 (类型保护是运行时检查)
if (isNumber(value)) {  
    let numberValue = value;  // TypeScript 现在知道 value 是 number 类型  
}

# 4. 类型保护

类型保护是运行时检查，有助于缩小条件块中变量的类型范围。它们允许函数和方法根据输入类型表现不同，而不会丢失类型信息，如typeof

// x: any：这表示函数接受一个类型为 any 的参数 x。any 类型是 TypeScript 中的一个顶级类型，它可以表示任何 JavaScript 值。
// x is string：这是类型谓词的声明。如果这个函数返回 true，那么参数 x 在那个上下文中应该被视为 string 类型。
function isString(x: any): x is string {  
  return typeof x === 'string';  
}

# 5. 类型断言的形式

类型断言是显式告诉编译器变量的类型

let someValue: any = "Hello World";  
let strLength1: number = (<string>someValue).length; // 尖括号形式的类型断言  
let strLength2: number = (someValue as string).length; // as形式的类型断言

# 6. TypeScript 中的接口（interface）及其用途

  • 接口用于定义对象的形状，描述对象应该具有的属性和方法。它强制了类型的一致性，有助于提高代码的可靠性和可理解性。

快速上手

  • 接口可以合并声明，即重复的接口可以合并
  • type可以声明联合类型，元祖

# 8. TypeScript 中的枚举（enum）类型是什么？

枚举是一种为一组数值赋予有意义名称的方式。它可以是数字枚举或字符串枚举。

enum Status {  
    Success = 200,  
    NotFound = 404,  
    ServerError = 500  
}

let currentDirection: Status = Status.Success;  // currentDirection 属于枚举类型，它的值为Status.Success
  
if (currentDirection === Status.Success) {  
    console.log("成功!");  
}

# 9. TypeScript 中定义具有可选参数和默认参数的函数

function greet(name: string, message: string = 'Hello', times?: number): void {  
  for (let i = 0; i < (times || 1); i++) {  
    console.log(`${message}, ${name}!`);  
  }  
}  
greet('John'); // Output: "Hello, John!"  
greet('Jane', 'Hi'); // Output: "Hi, Jane!"

# 泛型

泛型指的是在定义函数、接口或类时，不预先指定具体的类型，而是在使用的时候再指定类型限制。

在定义类、接口和函数时使用泛型参数，这些参数在尖括号（<>）中定义，并放在类型名称后面。
泛型参数在类、接口或函数内部可以代表任何类型，从而提供更高的代码灵活性和复用性。

类型参数化，运行时使用

  1. 泛型函数
function identity<T>(arg: T): T {  
    return arg;  
}  
  
// 使用泛型函数  
const number = identity<number>(42);    // 类型推断为 number  
const text = identity<string>("Hello"); // 类型推断为 string


// TypeScript 编译器通常可以自动推断类型参数，因此你可以省略类型参数：
const number = identity(42);    // 自动推断为 number  
const text = identity("Hello"); // 自动推断为 string

class GenericNumber<T> {

    constructor(zeroValue: T, add: (x: T, y: T) => T) {  
        this.zeroValue = zeroValue;  
        this.add = add;

console.log(myGenericNumber.add(1, 2)); // 输出 3
