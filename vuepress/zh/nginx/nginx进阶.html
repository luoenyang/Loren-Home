# 什么是nginx

Nginx是一个 轻量级/高性能的反向代理Web服务器。
高效的反向代理、负载平衡。官方支持5万左右的并发。
优点：反向代理服务器可以隐藏源服务器的存在和特征。

# 可以做什么用

  • 静态文件服务器
  • 反向代理服务器
  • 虚拟主机
  • 安全管理，对接口服务进行拦截

# 什么是正向代理

  • 代理端代理的是客户端，站在客户端那边为客户端说话;
  • 客户端给什么 代理端提交什么，得到什么取决于目标服务器，代理端基本不参与，一对一的关系。
  • 比如用nodejs做小后台（转发），
  • nodejs是后台语言不是服务器，但可以利用nodejs做中转服务，可正向代理也可反向代理 ;

# 什么是反向代理

  • 普通web代理服务器不支持外部对内部网络的访问请求，
  • 当一个代理服务器能够替代外部网络的主机访问内部网络时，这种代理称为反向代理。
  • 即可以指定一个地址为内部服务器地址的代理。
  • 代理端代理的是服务端，站在服务端这边，为服务端说话，对外屏蔽了服务器信息（保护作用）；
  • 并不是客户端给什么就提交什么，且客户端得到什么取决于在nginx的配置，一对多的关系；
  • nginx接受请求后去根据配置的规则转发到某个服务器，获取相应数据，然后将资源返回给客户端。
  • 客户端可以从一个或多个服务器获取资源（取决nginx的具体配置）。
  • 代理端对外表现为一个web服务器。

# nginx实现高并发原理

  • 每进来一个 request ，会有一个 worker 进程（调度进程）去处理，但不是全程处理；处理到可能堵塞的地方就注册一个事件，如果进程空闲了它回来接着处理完这个请求。
  • 就好像一个人在搬运东西，从A点（山上）搬到B点（山下），再从B点搬上车；车没来的时候，他会继续去A点搬到B点；而不是一直在B点等待。
  • 总结：
  • nignx采用单线程来异步非阻塞处理请求，不会为每个请求分配 cpu 和内存资源，节省了大量资源，同时也减少了大量的 CPU 的上下文切换。所以才使得 Nginx 支持更高的并发。
  • 和nodejs单线程几乎一个道理（事件驱动，事件循环，线程池）。

# 缓存配置 动静分离

动态文件与静态文件的分离。 静态文件做缓存。提高访问速度。
如果您在nginx中设置了静态目录，那么nginx会自动缓存静态文件，无需再进行额外的缓存配置。
nginx会将静态文件缓存到内存中，并且缓存时间为10秒。

location ^~ /static {
    # 如果您在Nginx中设置了静态目录，那么Nginx会自动缓存静态文件，无需再进行额外的缓存配置。
    # nginx会将静态文件缓存到内存中，并且缓存时间为10秒
    root $frontend_root;


    # 修改缓存配置

    # expires参数指定缓存时间，这里设置为1天
    expires 1d; 
    # expires 30s; # 缓存时间为30秒
    # expires 30m; # 缓存时间为30分钟
    # expires 3h; # 缓存时间为3小时

    # 设置为public，表示可以被任何缓存服务器缓存
    add_header Cache-Control "public";

    # 需要注意的是，如果您的静态文件经常更新，建议不要设置过长的缓存时间，否则可能会导致客户端无法获取最新的文件。

}

# 负载均衡配置

负载均衡实现的策略有以下常规3种，2种插件：

    1. 轮询(默认)：每个请求按时间顺序逐一分配到不同的后端服务器，如果后端某个服务器宕机，能自动剔除故障系统。
代码示例
upstream backserver { 
 server 192.168.0.12; 
 server 192.168.0.13; 
}

    1. 权重 weight：weight的值越大，分配到的访问概率越高
代码示例
# 权重越高，在被访问的概率越大，如下，分别是20%，80%。
upstream backserver { 
 server 192.168.0.12 weight=2; 
 server 192.168.0.13 weight=8; 
}

    1. ip_hash( IP绑定)：每个请求按访问IP的哈希结果分配
代码示例
ip_hash; 
 server 192.168.0.12:88; 
 server 192.168.0.13:80; 
}

    1. fair(第三方插件)
      必须安装upstream_fair模块。
      对比 weight、ip_hash更加智能的负载均衡算法，fair算法可以根据页面大小和加载时间长短智能地进行负载均衡，响应时间短的优先分配。
代码示例
# 哪个服务器的响应速度快，就将请求分配到那个服务器上。
upstream backserver { 
 server server1; 
 server server2; 
 fair; 
}

    1. url_hash(第三方插件)
      对标上面的ip_hash ， 必须安装Nginx的hash软件包 。
      按访问url的hash结果来分配请求，使每个url定向到同一个后端服务器，可以进一步提高后端缓存服务器的效率。
代码示例

 server squid1:3128; 
 server squid2:3128; 
 hash $request_uri; 
 hash_method crc32; 
}

# 内置的全局变量

ingnx有内置的全局变量，配置文件可直接用

  • $args： #这个变量等于请求行中的参数，同$query_string
  • $content_length： 请求头中的Content-length字段。
  • $content_type： 请求头中的Content-Type字段。
  • $document_root： 当前请求在root指令中指定的值。
  • $host： 请求主机头字段，否则为服务器名称。
  • $http_user_agent： 客户端agent信息
  • $http_cookie： 客户端cookie信息
  • $limit_rate： 这个变量可以限制连接速率。
  • $request_method： 客户端请求的动作，通常为GET或POST。
  • $remote_addr： 客户端的IP地址。
  • $remote_port： 客户端的端口。
  • $remote_user： 已经经过Auth Basic Module验证的用户名。
  • $request_filename： 当前请求的文件路径，由root或alias指令与URI请求生成。
  • $scheme： HTTP协议（如http，https）。
  • $server_protocol： 请求使用的协议，通常是HTTP/1.0或HTTP/1.1。
  • $server_addr： 服务器地址，在完成一次系统调用后可以确定这个值。
  • $server_name： 服务器名称。
  • $server_port： 请求到达服务器的端口号。
  • $request_uri： 包含请求参数的原始URI，不包含主机名，如：”/foo/bar.php?arg=baz”。
  • $uri： 不带请求参数的当前URI，$uri不包含主机名，如”/foo/bar.html”。
  • $document_uri： 与$uri相同。
