# 数据类型

查看

基础类型（值类型）：Number、String、Boolean、Null、Undefined、Symbol、bigInt
引用类型：object、Array、Date、Function、RegExp

  • 值类型：保存在栈中，占用空间固定、深复制
  • 引用类型：保存在堆中，可以保存无序，占用空间不固定，浅复制，可使用new() 方法构造出的引用型对象

深复制：两份数据在内存中是完全独立的，修改一个不影响另一个。
浅复制：引用类型的变量直接赋值实际上是传递引用，只是浅复制.是将变量中的存储的地址赋值一份单独存储，很常见的就是 Object、Array。

# 内存泄漏

应该被回收的内存没有被垃圾回收机制回收，内存没有被释放

  • 可能的原因：全局变量过大、定时器未清除、无限循环、dom清空时引用未清除
  • 可能的结果：变慢、崩溃、延迟大
// 1. 全局变量 aaaa 会一直暂时用内存 如果没有再被使用 会造成浪费
function aa() {
    aaaa = 123;
    console.log(aaaa)
}
aa();


// 2. 闭包
function createCounter() {
  let count = 0;
  return function() {
    count += 1;
    console.log(count);
  };
}
 
const counter = createCounter();
counter(); // 1
counter(); // 2
// 因这里 counter 函数仍然持有对 count 变量的引用，因此不会被垃圾收集器回收。可手动清除：counter = null;


// 3. 如果node不存在 定时器就不会被删除
let timer = setInterval(() => {
    const node = document.querySelector('#node');
    if (node) {
        clearInterval(timer);
    }
}, 1000)


// 4. DOM 元素引用，dom清空时引用未清除
// 如果一个对象（非DOM对象）包含对DOM元素的引用，那么这个DOM元素将不会被垃圾收集器回收。  
const myElement = document.getElementById('myElement');
const myObject = { element: myElement };
myElement.parentNode.removeChild(myElement);
// myObject 仍然持有对 myElement 的引用，导致 myElement 不能被回收。

# JavaScript 深拷贝浅拷贝

  • 浅拷贝：只复制了原对象或数组的第一层属性，对于引用类型属性只复制了引用。

  • 深拷贝：递归地复制了原对象或数组中的所有层级属性，创建了完全独立的副本。创建了一个新的对象或数组。

  • 内存地址特点：

浅拷贝会创建一个新的对象或数组，但并不会完全复制原对象或数组中的所有数据。
对于引用类型（如对象、数组等），浅拷贝仅复制引用，即新对象或数组的相应属性仍指向原对象或数组中的同一个内存地址。

深拷贝则相反，深拷贝出来的对象修改相关属性值不影响原来对象的值。

# this、new、和原型的关系 | 原型、原型链、继承

查看
  • 所有的函数都有prototype属性（原型）
  • 所有的实例对象都有__proto__属性 （指向创建它的引用类型的原型对象）
  • 而原型上有constructor属性 （指向构造函数）

原型链：当一个对象需要调用某个方法时，它会去最近的原型上查找该方法，如果没有找到，它会再次往下继续查找。这样逐级查找，一直找到了要找的方法。 这些查找的原型构成了该对象的原型链条。

继承：继承一个函数/对象的方法，减少代码量

# JS垃圾回收机制是怎样的

查看

js的垃圾回收机制是为了防止内存泄漏（已经不需要的某一块内存还一直存在着）， 垃圾回收机制就是不停歇的寻找这些不再使用的变量，并且释放掉它所指向的内存。
标记删除，进入执行环境时标记，离开执行环境再度标记，随后删除。

# 变量的生命周期

  • 全局变量：内存一直存在，直到浏览器/页面/组件关闭为止
  • 局部变量：作用域函数内部，当函数结束后，内存被释放

# js垃圾回收方式

  • 标记清除 (常用)

大部分浏览器使用这种垃圾回收，当变量进入执行环境(声明变量）的时候， 垃圾回收器将该变量进行了标记，当该变量离开环境的时候，将其再度标记，随之进行删除。

  • 引用计数

这种方式常常会引起内存的泄露，主要存在于低版本的浏览器。 它的机制就是跟踪某一个值得引用次数，当声明一个变量并且将一个引用类型 赋值给变量得时候引用次数加1，当这个变量指向其他一个时引用次数减1， 当为0时出发回收机制进行回收。

# vue在双向数据绑定是如何实现的

查看
  • vue双向数据绑定是通过数据劫持、组合、发布订阅模式的方式来实现的
  • 其核心是Object.defineProperty(obj, prop, descriptor)方法，对get和set进行重写操作

过程：
1.实现一个监听器Observer，用来劫持并监听所有属性，如果有变动的，就通知订阅者。
2.实现一个订阅者Watcher，可以收到属性的变化通知触发解析器Compile并执行相应的函数，从而更新视图。 3.实现一个解析器Compile，解析 Vue 模板指令，将模板中的变量都替换成数据。可以扫描和解析每个节点的相关指令，并根据初始化模板数据以及初始化相应的订阅器。

<template>
  <div>
    <!-- .stop 修饰符 -->
    <div @click.stop="handleClick" style="padding: 20px; background-color: #f0f0f0;">
      点击我（不会冒泡到外层）
      <button @click="innerButtonClick">内部按钮</button>
    </div>

    <!-- .prevent 修饰符 -->
    <a href="https://www.example.com" @click.prevent="handleLinkClick">点击链接（不会跳转）</a>

    <!-- .self 修饰符 -->
    <div @click.self="handleSelfClick" style="padding: 20px; background-color: #e0e0e0;">
      点击我或内部元素（只有点击外层 div 时才会触发）
      <span>内部元素</span>
    </div>

    <!-- .once 修饰符 -->
    <button @click.once="handleClickOnce">点击一次（之后不再触发）</button>
  </div>
</template>

<script>
export default {
  methods: {
    handleClick() {
      console.log('外层 div 被点击了');
    },
    innerButtonClick() {
      console.log('内部按钮被点击了');
    },
    handleLinkClick() {
      console.log('链接被点击了，但不会跳转');
    },
    handleSelfClick() {
      console.log('只有外层 div 被点击了，内部元素点击无效');
    },
    handleClickOnce() {
      console.log('按钮被点击了一次');
    }
  }
};
</script>

# vue.set(target,key,value) 方法原理

查看

向响应式对象添加属性

在两种情况下修改 Vue 是不会触发视图更新的(vue3没这种情况)

  • 在实例创建之后添加新的属性到实例上（给响应式对象新增属性）
  • 直接更改数组下标来修改数组的值

原理：当给对象新增不存在的属性，首先会把新的属性进行响应式跟踪 然后会触发对象 ob 的dep收集到的 watcher 去更新； 当修改数组索引时我们调用数组本身的 splice 方法去更新数组。

# 回流(重排)和重绘

查看

什么是回流？

  • 当一个元素自身的宽高，布局，显示或隐藏，或元素内部的文字结构发生变化，导致需要重新构建页面的时候，就产生了回流

什么是重绘？

  • 当一个元素自身的宽高，布局，及显示或隐藏没有改变， 而只是改变了元素的外观风格的时候，就产生了重绘

# get和post的区别

查看
  • 参数 GET把参数包含在URL中，POST通过request body传递参数
  • 缓存 GET请求会被浏览器主动cache，而POST不会，除非手动设置
  • 编码 GET请求只能进行url编码，而POST支持多种编码方式
  • 返回 GET在浏览器回退时是无害的，而POST会再次提交请求
  • 历史 GET请求参数会被完整保留在浏览器历史记录里
  • 长度 GET请求在URL中传送的参数是有长度限制的，而POST没有
  • 安全 GET比POST更不安全，因为参数直接暴露在URL上，所以不能用来传递敏感信息

# typeof与instanceof区别

  • typeof 检测基础数据类型操作符返回一个字符串
  • instanceof 返回一个true/false
  • instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象(new 创建出来的)的原型链上

let Car = function() {}
let benz = new Car()
benz instanceof Car // true

let car = new String('xxx')
car instanceof String // true

let str = 'xxx'
str instanceof String // false
