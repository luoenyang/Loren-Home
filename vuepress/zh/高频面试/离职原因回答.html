# 2. 原来公司我负责的方面

首先，我们的平台是低代码配置型的，分为开发环境和用户环境，开发环境是实施人员根据用户需求，通过相应的配置项配出一个系统，快速交付给用户。相比传统的开发系统交付更灵活、更快速。

灵活和快速的体现在：可通过配置配出各式各样的报表，查询项，自定义报表行按钮，自定义数据源；
工作流也是如此，保存成一个模板后在用户端就可以实时查看。这样就达到不需要动代码也可以快速实现变更需求的目的。

我负责的板块就是报表和工作流，报表也包括图表，用的是echarts做。
负责配置项的开发，保存成一个模板，最后在用户端根据相应的模板配置项渲染不同的报表工作流。
其次也做一个性化的项目，比如项目管理甘特图（并且实现从excel的导入导出）。

# 4. 关于vue的遇到过哪些技术难点，如何解决

  • iframe页面的嵌套，数据传输
    解决：通过postMessage进行通信
  • 小程序的文件上传（uploadFile只能使用post），但文件系统是minio（put）
    解决：用户选择文件后，通过临时路径把获取已buffer文件流的形式获取回来，再通过axios post进行上传； 但这里有个兼容封装传的注意点：就是用户可能是同手机微信访问小程序，也可能是通过PC端访问小程序；用户选择文件后我们得到临时url是不一样的，所以获取在获取buffer文件流的方式是不一样的，PC端可以使用axios，手机端要使用微信的api。
  • 路由传参，参数是由上个页面传入的（非query），刷新后参数没有了
    解决：通过缓存或者vuex，根据项目大小等实际情况使用
  • 页面缓存的问题，比如填写了一个表单，但底部有一个更新协议的按钮需要切到另一个组件，更新后返回 要原来的表单还在
    解决：使用vue的keep-alive保留组件状态避免重新渲染，或者 把表单数据进行缓存待提交后再清除缓存；两者都用过 按具体情况使用
  • 项目打包更新后的文件缓存问题，以及数据缓存问题
    解决：文件-文件的后缀加时间戳或者版本号；数据的缓存- 1. 把时间戳或者版本号存在前端变量或者vuex里；2.前后台结合
  • 大数据列表
    解决：虚拟列表，依据- 可视窗口 内容显示区域 虚拟区域，滚动的距离，数据行显示的开始下标和结束下标

# 5. vue有什么优势，对vue的理解，有什么优缺点

  • vue是渐进式Js框架，在核心库的基础上可以按需引入功能 比如vuex。
  • vue组件化开发，按需引入组件，简化了前端开发过程。
  • vue是通过数据驱动，当数据发生变化时，视图会自动更新，减少了对真实dom的操作。
  • vue实现了双向数据绑定，简少了对表单控件事件的监听操作。
