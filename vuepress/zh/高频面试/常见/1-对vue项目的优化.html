# 项目优化

  • 优化目的：快，如何才能快-网络顺畅和体积小
  • 可分两方面：网络加载，页面渲染（代码层级）

# 网络优化，加载优化

查看
  • 浏览器数据持久化localstorage
  • http的缓存：
    协商缓存：
    请求服务器 服务器判断是否需要变更（通过在请求头中包含一个条件标签来实现的）
    强缓存：
    缓存未过期前浏览器直接从本地缓存加载资源，而不需要向服务器发送请求
  • 多服务器 nginx配置负载均衡，把请求分配到不同的服务器
  • 压缩js,css
  • 样式表放在 link 中
  • js引入放在页面底部
  • defer 是在html解析完毕才执行,如果有多个则按加载顺序执行
  • async 是加载完毕后立即执行,如果是多个,执行顺序与加载顺序无关，执行时会阻塞元素渲染
  • 预加载 link rel="preload"
  • 避免重定向
  • 使用CDN
  • 提取公共代码
  • 使用 Chrome Performance 查找性能瓶颈

# 你都做过哪些 Vue 的性能优化（代码层级的优化）？

查看
  • 减少http请求，减少DNS查询
  • 减少DOM数量
  • 减少DOM操作
  • 减少iframe使用
  • 使用字体图标
  • 避免src为空
  • 对象层级不要过深，否则性能就会差
  • 不需要响应式的数据不要放在 data 中（微信小程序官方检测也如此）
  • v-if 和 v-show 区分使用场景
  • computed 和 watch 区分场景使用
  • v-for 遍历必须加 key，key最好是id值，且避免同时使用 v-if
  • 大数据列表和表格性能优化 - 虚拟列表 / 虚拟表
  • 防止内部泄露，组件销毁后把全局变量和时间销毁（定时器等）
  • 图片懒加载
  • 路由懒加载 按需引入
  • 适当采用 keep-alive 缓存组件
  • 防抖、节流的运用
  • 服务端渲染 SSR or 预渲染
