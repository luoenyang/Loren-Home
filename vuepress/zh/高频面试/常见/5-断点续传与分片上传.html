# 断点续传

主要原理：

  • 客户端请求头 Range ：告诉服务端从哪里开始下载/上传
  • 服务端响应头 Content-Range 、 Content-Length ：返回从哪里开始接收/返回、总大小
  • 可自行组装blob 再进行下载。 可参考Axios下载

# 分片上传与合并

在客户端将整个文件分隔成多个数据块来进行分别上传，上传完之后再由服务端对所有上传的文件进行汇总整合成原始的文件。

  • 前端自行分片原生文件API： file.slice(start,end) 进行切片
  • 前端分片插件：百度 webuploaderopen in new window 首页open in new window
  • 后端合并与写入：两种工具：RandomAccessFile、MappedByteBuffer

前端切片封装示例

// 很好解释了 generator（不会立即执行，可循环自动执行），yield（暂停执行 交出执行权，等待结果返回）



const MB=1024*1024;

// 切片方法
function* slice(file){
    let i=0;
    let start=0;
    while(true){
        // 这里规定每片1MB，最后一片小于1MB
        const end=start+MB;
        const blob=file.slice(start,end);
        if(blod.size==0){
            // 跳出循环体
            break;
        }
        yield {chunk:++i,blob}
        start=end;
    }
}

// generator函数返回的是generator对象， 满足迭代器协议，可以使用 for of 进行遍历。不需要.next()
// 所以 slicer 中 其实包含所有的 { chunk, blob } 切片信息。
let slicer=slice(file);
for(const slice of slicer){
    // slice 包含返回的{chunk,blob}
    console.log(slice)
}
