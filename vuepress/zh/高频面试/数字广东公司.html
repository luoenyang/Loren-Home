# 1. 盒子模型有几种，有什么区别

查看
# box-sizing 改变 盒模型类型：

标准盒子模型由内容（content）、内边距（padding）、边框（border）和外边距（margin）四个部分组成。
在标准盒子模型中，元素的宽度（width）和高度（height）仅包括内容区域的尺寸。

  • content-box（默认值），盒子定义宽高 = 内容宽高（content）
  • border-box（IE） ，盒子定义宽高 = 内容宽高（content） + 内边距 （padding）+ 边框（border）
  • padding-box ， 盒子定义宽高 = 内容宽高（content） + 内边距 （padding）
# display 定义盒子布局 盒子布局的2种方式
  • box 布局子元素不是1:1的时候分不均匀, 如2:1时,真实的不是2:1
  • flex 布局是均分、等比分。 flex布局

# 3. 什么是BFC？如何触发BFC？

查看

什么是BFC？

  • BFC(Block formattin context)， 直译：“块级格式化上下文”。是一个独立的渲染区域，不受外界干扰，外界也不受其里面的布局干扰。

如何触发BFC？

  • position为absolute或fixed;
  • float属性不为none;
  • overflow不为visible;
  • display为inline-block，table-cell，table-caption，flex， inline-flex

应用：
BFC在CSS布局中有广泛应用，如防止外边距折叠、清除浮动等。

# 4. 如何清除浮动

查看

为什么要清除浮动？

  • 当容器的高度为auto ，容器的内容中有浮动（float为left或right）的元素，在这种情况下，容器的高度不能自动伸长以适应内容的高度，使得内容溢出到容器外面而影响（甚至破坏）布局的现象。这个现象叫浮动溢出，为了防止这个现象的出现而进行的CSS处理，就叫CSS清除浮动。

如何清除？

  • 在父级内放置一个空标签，设置样式 clear:both 清除浮动
  • 在父级设置伪元素:after 设置样式 clear:both 清除浮动
  • 设置父级overflow:hidden/auto/scroll

什么是圣杯布局？

  • 圣杯布局是两边固定宽度，中间自适应的三栏布局。
  • 中间栏放到文档流前面，先行渲染。
  • 目的是为了解决三栏布局。

操作：

  • 左右分别浮动向左、右
  • 中间设置100%宽，在设置padding这margin即可
  • 最后记得清除浮动，否则浮动元素会溢出

# 8. position

查看

position 属性的五个值：

  • static 默认定位，没有定位，随文档流显示
  • relative 相对定位 - 相对其正常位置，可设置上下左右位置、边距
  • fixed 固定定位 - 相对浏览器窗口、视窗的定位

  • sticky 粘性定位 - position: sticky;top: 0;滚动到它时"粘在顶部"

# 14. http比较常用的请求方式 http

查看

RESTful接口

  • get、post、put、delete、head、options、trace、connect
  • GET 获取
  • POST 创建
  • PUT 全部更新
  • PATCH 局部更新
  • DELETE 删除
# get、post的区别
  • 参数 GET把参数包含在URL中，POST通过request body传递参数
  • 缓存 GET请求会被浏览器主动cache，而POST不会，除非手动设置
  • 编码 GET请求只能进行url编码，而POST支持多种编码方式
  • 返回 GET在浏览器回退时是无害的，而POST会再次提交请求
  • 历史 GET请求参数会被完整保留在浏览器历史记录里
  • 长度 GET请求在URL中传送的参数是有长度限制的，而POST没有
  • 安全 GET比POST更不安全，因为参数直接暴露在URL上，所以不能用来传递敏感信息

function unique(arr) {
	for (let i=0; i < arr.length; i++) {
  	for (let j=i+1; j < arr.length; j++) {
    	if (arr[i] == arr[j]) {
      	arr.splice(j, 1);
        j--;
      }
    }

  4. 网络请求不会404错误，可以配置vue路由404页面。
  5. onhashchange事件，可以在windows对象上监听这个事件
